# Известные баги

## BUG-001 — часть чатов не отображается (лимит диалогов)

**Симптом**
В папке «Personal» видны не все личные чаты; этих же чатов нет и в общем
списке чатов.

**Как воспроизвести**
Иметь больше ~50 диалогов. Чаты, не попавшие в топ-50 по последней активности,
отсутствуют и в общем списке, и в папках.

**Вероятная причина**
`getChats` тянет только `DIALOG_LIMIT = 50` диалогов
([TelegramProvider.ts:24](../src/providers/telegram/TelegramProvider.ts#L24),
[:165](../src/providers/telegram/TelegramProvider.ts#L165)). Папки
(`getFolders`) классифицируют **только уже загруженные** чаты
([:436](../src/providers/telegram/TelegramProvider.ts#L436)), поэтому чат из
`includePeers` папки, не попавший в топ-50, не добавляется в `folder.chatIds`.
Итог — чат пропадает и из общего списка, и из папки.

**Куда копать при фиксе**
- увеличить лимит / постранично догружать все диалоги (`getDialogs` с итерацией);
- проверить архивные диалоги (`getDialogs` может их не включать — нужен
  `archived`/folder 1);
- папки должны резолвить `includePeers`/`pinnedPeers` даже для чатов вне топ-50
  (подтягивать недостающие сущности).

**Исправлено:** `DIALOG_LIMIT` поднят 50 → 1000
([TelegramProvider.ts:24](../src/providers/telegram/TelegramProvider.ts#L24)),
`getDialogs` подтягивает все диалоги, папки классифицируют полный список.

**Архив:** уточнение — отдельный запрос `archived` не нужен. GramJS при
неуказанном `folder` возвращает и архивные диалоги (у них `dialog.archived ===
true`, `folderId === 1`); раньше они просто подмешивались в общий список. Теперь
архивные помечаются флагом `Chat.archived`, исключаются из «Все чаты» и
кастомных папок и выводятся отдельной папкой **«Архив»** (иконка `archive`) в
конце дерева — как в Telegram.

**Остаётся (edge cases):** аккаунты с >1000 диалогов (пагинация `getChats`
завязана на путь refresh — отложено, чтобы не бить FLOOD_WAIT на каждое
сообщение) и резолв `includePeers` папки для чатов вне выборки. Пока не
встречалось — вернёмся при необходимости.

**Статус:** исправлен (осн. случай + архив отдельной папкой).

## BUG-002 — уведомления приходят по замьюченным чатам

**Симптом**
Toast-уведомления о новых сообщениях прилетают даже для чатов, у которых в
Telegram отключены уведомления (mute).

**Вероятная причина**
`maybeNotify` ([extension.ts](../src/extension.ts)) не учитывает mute: показывает
toast для любого входящего (не своё, включены уведомления, чат не открыт). Статус
mute из Telegram нигде не читается.

**Исправлено:** провайдер держит живую карту `mutedChats` — заполняется из
`getChats` (`notifySettings.muteUntil`), обновляется в реальном времени по
`UpdateNotifySettings` (mute/unmute с любого клиента) и при своём `setMuted`.
`maybeNotify` спрашивает `provider.isChatMuted(chatId)` напрямую (не через
дерево, которое могло быть устаревшим/несмонтированным).

**Остаётся (мелочь):** если чат вообще не попал в загруженные диалоги, mute
неизвестен → уведомление пройдёт (тот же edge, что у BUG-001).

**Статус:** исправлен.
