## Реакции (провайдеро-агностично) — ДО следующего провайдера

Реакции (эмодзи под сообщением) — общая фича, а не Discord-only. Делаем её
**перед подключением нового провайдера** (Slack/Teams), чтобы общая модель и UI
существовали до маппинга 4-го бэкенда, а не допиливались задним числом.

Все три текущие либы это умеют (проверено):

- **Discord**: `message.react(emoji)`, `ReactionManager` на сообщении, гейтвей
  `messageReactionAdd`/`messageReactionRemove`;
- **Telegram**: `Api.messages.SendReaction`, реакции приходят в сообщении;
- **WhatsApp**: `sock.sendMessage(jid, { react: { text, key } })`.

Из чего состоит:

- **модель**: `Message.reactions?: { emoji, count, me }[]`;
- **интерфейс**: `addReaction`/`removeReaction`; realtime — переиспользовать
  `onMessageEdited` (реакция меняется → перевыпуск сообщения с обновлёнными
  реакциями, webview перерисует; новое событие не нужно);
- **провайдеры**: маппинг входящих реакций в `toMessage`, отправка/снятие,
  подписка на reaction-события;
- **webview**: чипы реакций (эмодзи + счётчик, подсветка своих), клик = тоггл,
  добавление через быстрый набор (👍❤️😂🎉 по ховеру рядом с ↩), не полный пикер.

Тиры (инкрементально, как mute):

- **Тир 1 — read-only**: показ чипов (входящие + realtime), без добавления;
- **Тир 2 — + отправка**: клик-тоггл своих + быстрый набор эмодзи.

Границы старта: только юникод-эмодзи; кастом-эмодзи Discord (`<:name:id>`) —
как `:name:`, картинки позже.
